# CLAUDE.md

本项目是 `tdl`（Telegram CLI 工具）的自建分支，主要新增了网页版（web UI）。当前目录 `tdl-webui` 是活动开发副本（git 仓库，含 go.mod）。

## 先读项目笔记

动手前先看 [`项目笔记/README.md`](项目笔记/README.md)——它是按功能块拆分的索引，能让你只读相关章节而不必通读代码。常用对应：上传→`05-上传.md`、网页版→`06-网页版.md`、下载→`03-下载.md`、带宽限速→`08-带宽限速.md`。

## 改动代码后，更新对应笔记

完成代码改动后，把要点更新到 `项目笔记/` 里对应的主题文件（改了哪块就更哪个文件）。规则：

- **只记代码和 git 看不出来的东西**：踩过的坑、为什么这么选（而非另一种方案）、用法/命令、跨模块约束。
- **不要复述代码**：函数名、代码结构、改了哪几行——这些 git 和源码本身就有，记了只会冗余、徒增以后读取的上下文。
- 判断标准：三个月后忘了细节，这条能帮人/AI 快速捡回来吗？能才记。
- 只有在**新增/删除主题文件或某文件覆盖范围变化**时才更新 `项目笔记/README.md` 的索引表；改内容不用动索引。

## 工程结构要点（动手前须知）

- 多 module 工作区（`go.work`：根 `.`、`core`、`extension`、`docs` 各有独立 go.mod）。`core` 是独立模块 `github.com/iyear/tdl/core`，根模块依赖它，**所以 `core` 不能反向 import 根的 `pkg/*`**（否则模块循环）。给 core 加第三方依赖要在 `core/go.mod` 里单独加。
- 验证编译：根模块 `go build ./...`；`core`、`extension` 各自目录下再 `go build ./...`（go.work 已串起来）。
- 前端是 `go:embed` 进二进制的，改 `app/web/ui/` 下文件后要重新 `go run . server` 才生效，不热更新。
- 没装 tdl 二进制，命令前缀是 `go run .`（不是 `tdl`）。
