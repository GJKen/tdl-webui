# 05 · 上传

## 操作笔记

### 注意：命令前缀是 `go run .` 而不是 `tdl`
没装 tdl 二进制，命令前缀是 `go run .`。
（想直接用 `tdl` 命令就先构建：`go build -o tdl.exe .`，之后用 `.\tdl.exe up ...`）

### 发到 Saved Messages（发给自己，最安全的测试）
```
go run . up -p "文件或目录路径"
```

### 发到指定群组 / 频道
```
go run . up -p "文件" -c 目标
```
- 目标可以是【用户名】(公开频道/群，如 `-c durov`) 或【数字ID】(私有的)。
- 数字ID 从 `go run . chat ls` 里查（表格里的 ID 是 raw id，不带 `-100` 前缀）。
- 私有目标第一次用前先跑一遍 `chat ls`，把 peer 缓存下来，否则按 id 解析不到。
- 话题群再加 `--topic 话题ID`（话题ID 也在 `chat ls` 的 Topics 列里）。
- 要对目标有发送权限（频道要管理员发帖权，群要在群里且能发媒体）。

### 查看可上传的目标
```
go run . chat ls                                  # 全部会话
go run . chat ls -f "Type == 'channel'"           # 只看频道
go run . chat ls -f "Type == 'group'"             # 只看群组
```

### caption（上传后消息底下的说明文字）
默认【不是空的】！不指定时会自动带上「文件名(去扩展名) - MIME类型」，渲染成等宽代码样式，例如：
```
[AI-Raws] 天空戦記シュラト #01 (BD HEVC 1440x1080 FLAC)[45ABB24A] - video/x-matroska
```
`--caption` 收的是【expr 表达式】不是纯文本，注意引号嵌套（外层单引号给 PowerShell，内层双引号给 expr 当字符串）：
- 不要说明：     `--caption '""'`          ← 见下方踩坑，PowerShell 下只能这样写
- 只要文件名：   `--caption 'FileName'`
- 文件名带后缀： `--caption 'FileName + FileExt'`
- 固定文本：     `--caption '"我的说明"'`

可用字段：`FilePath` / `FileName`(无扩展名) / `FileExt` / `ThumbPath` / `MIME`
看全部字段：     `go run . up --caption -`

### 踩过的坑
- 【`-c` 前面一定要有空格】。写成 `"xxx.mkv"-c 1234` 会被 PowerShell 拼成一个参数，路径变成 `xxx.mkv-c`、`-c` 失效。正确写法：`... .mkv" -c 1234`
- 路径带空格 / 特殊字符（`[ ] # ! ( )`）要整体用双引号括起来。tdl 用 WalkDir 遍历、不做通配符展开，brackets 按字面处理，没问题。
- 【清空 caption 只能写 `--caption '""'`】(单引号里包两个双引号)。PowerShell 5.1 会把空参数直接吞掉：`--caption ""` 和 `--caption ''` 都会让 tdl 报 `flag needs an argument: --caption`。而 `'""'` 会被当成空字符串传进去，正好得到空 caption。
- 单文件上限：普通号 2GB，Premium 4GB。
- 示例（把 ~1GB 的 mkv 发到 id 为 1202784229 的频道）：
  ```
  go run . up -p "D:\!Amine\!Watching\[AI-Raws][Shurato]\[AI-Raws] 天空戦記シュラト #01 (BD HEVC 1440x1080 FLAC)[45ABB24A].mkv" -c 1202784229 --caption '""'
  ```

> 上传也受全局 `--rate` 限速，见 [08-带宽限速.md](08-带宽限速.md)。网页端上传见 [06-网页版.md](06-网页版.md)。

## 架构与实现

上传逻辑在 `app/up/up.go` 和 `core/uploader`。流程包括：扫描本地文件 → 按 include/exclude 过滤 → 解析目标 peer → 编译 caption 表达式 → 创建上传迭代器和进度条 → 并发上传到 Telegram。上传侧支持表达式能力，例如根据文件名、扩展名、MIME 类型生成目标或 caption。

目标 peer 的指定方式（见 `cmd/up.go`）：

- `-c/--chat`：聊天 id 或用户名，可配合 `--topic` 指定话题群中的话题；为空表示发送到 “Saved Messages”（自己）。
- `--to`：基于表达式引擎的路由，可按文件属性把不同文件分发到不同目标，与 `--chat`/`--topic` 互斥。
- 另有 `--photo`（以图片发送）、`--file`（强制按普通文档发送，与 `--photo` 互斥）、`--rm`（上传后删除本地文件）、`--album`（把文件发成媒体组/相册，普通号每组 ≤10，按顺序）。
- 相册模式（`--album` 或网页相册）走 `app/up` 的 `runAlbum`：按顺序上传组内文件、caption 只放第一项、每 ≤10 个用 gotd 的 `message...Album()` 发成一条相册消息（>10 自动分多组）。`core/uploader` 把「上传文件 + 构建媒体」抽成可复用的 `BuildMedia`，单发与相册共用。
- 逐文件发送（`core/uploader/uploader.go` 的 `Upload`）对单个文件的发送错误不会中断整批（除用户取消外），但真实错误会先传给进度回调 `OnDone` 再决定是否继续——CLI 端据此把该文件进度条标红并打日志，网页端据此把任务标 `error`。
- `--caption`：上传后消息的说明文字，本质是 expr 表达式（不是纯文本），对每个文件求值得到字符串后再按 HTML 解析。默认值是表达式 `"<code>"+FileName+"</code> - <code>"+MIME+"</code>"`，所以不指定时默认 caption 是「文件名（去扩展名）- MIME 类型」并渲染成等宽样式。可用字段见 `Env`（`app/up/up.go`）：`FilePath`、`FileName`、`FileExt`、`ThumbPath`、`MIME`；运行 `up --caption -` 可打印全部字段。空表达式即得到空 caption。

peer 解析在 `core/util/tutil/tutil.go` 的 `GetInputPeer`：非数字按用户名解析（`manager.Resolve`），数字按 id 依次尝试 channel/user/chat。私有目标通常需先经 `chat ls` 等操作把 access hash 写入 peer 缓存，才能按 id 解析。`-p` 指定的路径由 `filepath.WalkDir` 遍历（`app/up/walk.go`），不做通配符展开。

### 聊天工具（chat）
`cmd/chat.go` 提供三个子命令：`chat ls`（列出聊天）、`chat export`（导出聊天消息）、`chat users`（导出频道/群成员）。这些工具服务于下载和转发流程，例如先导出消息 JSON，再批量下载或转发。
