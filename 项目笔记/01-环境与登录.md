# 01 · 环境与登录

## 操作笔记

### 安装 Go 语言环境
等待 download 下载完成，会出现下面的字样（文字在 PowerShell 里看不清，要手动复制）：

```
Version: dev
Commit: unknown
Date: unknown

go1.26.4 windows/amd64
```

### 运行登录（首次报错及修复）
```
go run . login
```
提示报错：
```
WARN: If data exists in the namespace, data will be overwritten
Error: no data found in possible paths, please specify path to Telegram Desktop directory with -d flag
exit status 1
```
解决：
```
go run . login -d "E:\Program\AyuGram\tdata"
```

### 我的第三方 TG 安装路径
- 路径：`E:\Program\AyuGram`
- AyuGram 的 tdata 在 `E:\Program\AyuGram\tdata`（绿色便携版，数据在程序目录），是标准 tdesktop 格式。

### 登录方式总结（同一个账号，三种登录，session 各自独立）

**1. 桌面端导入（`-d`，就是上面那条报错的修复）**
```
go run . login -d "E:\Program\AyuGram\tdata"
```
- 直接复用 AyuGram 已有 session，不用手机验证码。
- 【前提：必须先完全退出 AyuGram】，否则 tdata 文件被占用，读取失败。
- 和 AyuGram 共用同一把 auth key，两边同时高强度使用偶尔会互相挤下线。
- 导入时会问 “是否登出桌面端 session”，想继续用 AyuGram 就选 No。

**2. 扫码登录（Windows 终端扫不动，放弃）**
```
go run . login -T qr
```
- 终端用半块字符画二维码，PowerShell 里渲染会错位，基本扫不了。

**3. 验证码登录（★最终实际用的就是这个，最稳）**
```
go run . login -T code
```
- `Enter your phone number:` 默认值是 `+86 12345678900`，要清空重输自己的号，格式 国家码+号码，例如 `+86 13800138000`。
- 验证码会发到【已登录的 AyuGram】里（官方 "Telegram" 账号发来的消息），不是短信。
- 开了两步验证会再问 `Enter 2FA Password:`（输入不显示，正常）。
- 成功显示：`Login successfully! ID: xxxxxxx, Username: xxxx`

> 当前登录状态、session 存储位置见 [07-存储备份迁移.md](07-存储备份迁移.md)。代理相关见 [02-网络与代理.md](02-网络与代理.md)。

## 架构与实现

登录命令定义在 `cmd/login.go`，业务在 `app/login`。支持三种登录方式（通过 `-T` 指定）：

- `desktop`（默认）：从 Telegram Desktop 或其兼容分支（如 AyuGram）的 `tdata` 目录导入已有 session。核心在 `app/login/desktop.go`，使用 `gotd/td` 的 `session/tdesktop` 解析 tdata；`-d` 指定目录（为空则自动探测常见安装路径），`-p` 传桌面端本地密码（passcode）。导入后可选择登出桌面端 session，以避免冲突。
- `code`：`app/login/code.go`，在终端输入手机号与 Telegram 下发的验证码（已登录的客户端会收到，而非短信），若开启两步验证则再输入密码。
- `qr`：`app/login/qr.go`，终端渲染二维码，用手机 Telegram 扫码关联桌面设备。

三种方式都会把 session 写入当前 namespace 的 KV 存储，并标记 app 类型为 desktop。登录的是同一个 Telegram 账号，但不同登录方式或不同 namespace 得到的是相互独立的 session。
