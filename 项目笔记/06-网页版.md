# 06 · 网页版（web UI，自建功能）

CLI 之外加了个本地网页服务，用来【多账号登录】+【指定文件传到指定目标】。代码在 `app/web/`，命令是 `server`（别名 `web`），已注册进 `cmd/root.go`。这是在原 `tdl` 之上本地自建的功能，并非上游原生能力。

## 操作笔记

### 启动
【先确认目录】`cd D:\Study\Python\tdl-webui`（这里有 go.mod。）
```
go run . server                 # 默认 http://127.0.0.1:8080
go run . server -P 8099         # 换端口
go run . server --host 0.0.0.0  # 【不建议】对外暴露，没有鉴权
```
- 要代理就照常加 `--proxy`；OpenClash 透明代理不用加。
- 网页设置弹窗里的代理会持久化，重启后优先于启动时的 `--proxy`；保存新代理会断开所有已连接账号，下次操作按新代理重连。
- 启动后终端会打印访问地址，浏览器打开即可。`Ctrl+C` 正常退出。
- 改了 `app/web/ui/` 下的网页文件要重新 `go run`（前端是 `go:embed` 编译进二进制的，不热更新）。静态文件已设 no-cache，重新 `go run` 后普通刷新即可，一般不用强刷（首次从旧缓存切过来可能仍需 `Ctrl+R` 一次）。
- 想用 `tdl server` 而不是 `go run`：先 `go build -o tdl.exe .`，之后 `.\tdl.exe server`。

### 顶部账号栏（2026-06-16 改版）
账号管理从原来的“账号”标签页挪到了顶部：左上角是【当前账号按钮】，右上角是 "tdl web UI" 字样。
- 点左上账号按钮 → 弹出下拉（带淡入动画）：
  - 列出所有已登录账号（= 各 namespace），点某一行就【切换】为当前账号并高亮。
  - 每行右边三个按钮：⟳ 查看/刷新（真连一次，显示 ID + 昵称并缓存）、✎ 改名、🗑 删除。
  - 底部“＋ 添加账号 / 登录”按钮 → 弹出登录窗。没有任何账号时，下拉里就只有这个登录按钮。
- 左上按钮上：上面一行是名字（别名/昵称），下面一行只显示 ID（名字不再重复出现在 ID 那行）；上传用的就是这个当前账号。

#### 改名 = 别名（不动 namespace）
- ✎ 改的是【显示别名】，存在该账号自己的 KV 里（key=`web:alias`），【底层 namespace 不变】。
- 所以把 default 改成“我的主号”后，CLI 仍然照常：`go run . up ...` 默认还是 `-n default`，不受影响。
- 别名留空 = 恢复显示成 namespace 原名。

#### 删除账号
- 🗑 → 确认 → 真正删掉该 namespace 的本地数据（bolt 文件），不可恢复（相当于登出 + 清 session）。
- 删除时会先断开该账号的常驻连接，再删文件（否则 Windows 上文件被占用删不掉）。
- 删完【不会再冒出 default 占位账号】：账号列表只显示有登录态（session）的 namespace，已注销账号也不显示。

#### 添加账号 / 登录（弹窗向导）
1. 填【账号名称】+ 手机号 → 发送验证码。
   - 账号名称【可留空】：留空就用登录后取到的 Telegram 昵称当名字；填了就用你填的。它同时用作 namespace（非法字符 `/ \ : * ? " < > |` 会自动换成 `_`）。
   - 手机号【不用手打 +】：输入框已固定带 + 前缀，直接填「国家码+号码」，例如 `86 13800138000`（你手贱多打个 + 也会被规整掉）。
2. 填验证码（发到已登录的 TG 客户端，不是短信）。
3. 开了两步验证再填 2FA 密码。
4. 成功后自动设为当前账号。
- 提交验证码 / 密码后显示「校验中…」直到成功或失败，不会再来回跳文字。
- 登录中途点弹窗外面或按 Esc，会先问「放弃这次登录？」防手误；右上角 ✕ 是直接关。
- 改名 / 删除 / 报错都用网页内置弹窗（不是浏览器自带那种），删除是带红色按钮的确认框。

### 会话控制台（2026-06-16 再次改版：下拉框 → Telegram 式左右两栏）
原来“上传”标签是个下拉框选目标，已改成 Telegram 风格的左右两栏控制台（标签也改名“会话”），整页自适应铺满浏览器。

#### 左侧：会话列表
- 顶部搜索框：只过滤已缓存的会话；列表里没匹配到就显示“没有匹配的会话”，不再提供“手填 id/@用户名 直接发”的入口（该功能已移除）。
- 每行：彩色首字母头像（颜色随机生成、按会话存浏览器 localStorage 固定不变）+ 名字 + 右上角【最后一次发送时间】（今天 H:MM，否则 M月D日；没发过不显示）+ 下方【最后一次发送的文件名预览】（别的账号发的前面带“昵称:”；没发过不显示）。
- Saved Messages 永远置顶；右键会话弹菜单：置顶/取消置顶、复制 ID、刷新会话列表。
- 左下角 ⚙ 打开服务器级设置：上传限速、代理、代理连通性测试；下载限速项先占位禁用。
- 已注销账号（TG 标记 Deleted）不再出现在列表里（chat ls 同样过滤）。
- 列表 / 窗口只在内容真变化时才重渲染（避免 F12 里被每秒重建打扰）。

#### 设置弹窗（上传限速 / 代理）
- 上传限速是 server 全局设置，不按账号区分；留空表示不限速。它会持久化，且重启后优先于启动命令里的 `--rate`。正在上传的大文件会跟随新速率即时变化，因为 web 上传共用同一个可热更新 limiter 指针。
- 代理设置也是 server 全局设置，保存为 URL 语义：空值=直连，界面拆成协议（socks5/http）、地址、端口三块。当前拆分输入主要面向 `socks5://host:port` / `http://host:port`，带用户名密码的代理回填会不完整；真要用鉴权代理，需要再放宽解析。
- 保存代理不会只改新请求：它会把代理写回运行时配置并断开所有常驻账号连接，之后下一次使用账号时按新代理重连。代价是进行中的上传会被中断，所以界面会提示。
- “测试”按钮只用当前填写的代理（不需要先保存）尝试 TCP 连 Telegram 生产 DC；它验证的是网络可达，不做 MTProto 握手、账号鉴权或真实发送测试。
- 弹窗底部「数据备份 / 恢复」区：「备份」下载 `.tdl`、「恢复」上传 `.tdl` 合并导入，接口 `GET /api/backup`、`POST /api/recover`，复用 CLI 那套 backup/recover 核心（产物通用）。**恢复是合并覆盖不是整盘替换、备份文件未加密**，语义与坑见 [07-存储备份迁移.md](07-存储备份迁移.md)。

#### 右侧：会话窗口（点左侧会话打开；没选时空白）
- 顶部标题栏：头像 + 名字 + 类型/用户名/id + ⟳ 刷新会话列表。
- 中间是“只发不收的聊天记录”：每个上传任务一个圆角气泡——文件名、caption（有才显示）、左下角文件大小、右下角时间 + 状态（✓ 成功 / ⚠ 失败）。按日期分隔（M月D日）。
- 【自己当前账号发的靠右、别的账号发的靠左】；靠左的带该账号圆头像 + 昵称，同一人连续消息只在最上方显示一次名字。
- 发送中：气泡内有【旋转的进度圆环】，弧随上传进度增长（平滑过渡，不再抽搐），中间 ✕ 点击【取消上传】，取消后这条消息消失。圈在气泡内（自己发的靠左、别人发的靠右）。
- 相册显示成【一个气泡】：📚 相册 · N 个文件 + 文件名列表 + 总大小，配一个聚合进度环和取消（取消整组）。

#### 怎么发文件（两种）
1. 【拖文件进窗口】或点底部长条按钮【选择文件】 → 弹选项框：
   - 勾【作为相册】（多文件默认勾上）→ 整组发成一条媒体组（图片→照片、视频→视频；≤10/组，超出自动分组），会话里显示成一个相册气泡。
   - 不勾 → 逐个发，mp4/jpg/png 等可逐个选“作为照片/视频/音频”或“作为文件”。
   - 填共享 caption → 发送（按上到下顺序）。文件字节经浏览器传到服务器临时目录暂存，发完自动删。
2. 折叠的【服务器本地路径】模式：服务器本机上的大文件 / 目录批量，不经浏览器来回传，沿用 walk + include/exclude + 图片作照片。
- caption 现在是 **markdown**（不是 CLI 那套 expr）：`代码` 等宽、[文字](链接) 超链接、**粗** *斜* ~~删~~；留空 = 无 caption。

#### 混合相册冲突修复（2026-06-16）
- 【根因】相册模式服务端把每个文件塞进一个 `message.Album(...)` 一组发出。Telegram 不允许同一媒体组里混放「照片/视频」和「普通文件(文档)」，所以「图片 + 不能进相册的文件」必然在 `album.go:145` 报 send album 错。
- 【检测】前端把文件分三类——视觉(图/视频)、音频、文档；跨类即判为冲突（通用，不只图片+文件，音频混视觉、音频混文档也触发）。
- 【方案：勾选框仅在混合冲突时出现】
  - 不冲突：照常显示「作为相册」勾选。
  - 冲突：隐藏「作为相册」，改显示【全部作为文件发送（合并成一个相册）；取消勾选则分开逐个发送】，默认勾上。
  - 勾上 → 走相册接口并带 `as_file=true`：服务端每个文件 `AsFile:true / Photo:false`，当纯文档上传 → 全文档相册对 Telegram 合法（仍按 ≤10 自动分组）。
  - 取消 → 落回原来的逐个发送循环。
  - 增删文件时实时重判冲突。
- 【改了哪些文件】
  - `app/web/upload.go` — `handleUploadAlbum` 新增读取 `as_file` 字段，为真时 `AsFile:true / Photo:false`。
  - `app/web/ui/index.html` — 发送框加默认隐藏的「全部作为文件发送」勾选项；给原「作为相册」一行加 id 方便切换。
  - `app/web/ui/app.js` — 加 `albumCategory` / `hasAlbumConflict` 判定，按冲突切换两个勾选项的显隐与发送路径。
- 构建通过（`go build ./app/web/` 退出码 0）。前端是 `go:embed` 进二进制的，改完重新 `go run . server` 生效。

#### 几处修复 / 改进（2026-06-16）
1. 【收藏夹重复】刷新会话列表后，会多出一个以自己昵称命名的会话，和「Saved Messages」都指向收藏夹（收藏夹有内容时才会冒出来）。
   - 根因：Telegram 把「自己」当成一个普通私聊返回；而 web 永远会再注入一个合成的「Saved Messages」→ 同一目标出现两行。
   - 修复：`app/web/chats.go` 过滤掉「自己」那条私聊（`selfPeerID` 优先读缓存的 self id，没有就 `c.Self` 兜底并回写缓存；`dropSelfDialog` 删掉 type=private 且 id==self 的那条）。刷新路径过滤后再写缓存，旧缓存在下次读取时也会被补过滤。
2. 【没权限却显示成功】发到没有发送权限的频道，webUI 和终端都显示成功——不对。
   - 根因：`core/uploader/uploader.go` 的逐文件路径把「非取消类」的发送错误吞掉了（OnDone 之前就 return nil），所以 `CHAT_SEND_MEDIA_FORBIDDEN` 之类被当成功。相册路径本来就会把错误抛出，所以只有逐个发的单文件路径有这个 bug。
   - 修复：先拿到真实错误再传给 OnDone（CLI 端会把进度条标红 + 打日志；web 端 `app/web/progress.go` 的 OnDone 把任务标成 error）。非取消错误仍然不中断整批（多文件批量时一个坏文件不连累其它）。`app/web/upload.go` 收尾时若任务已被标 error 就不再覆盖成 done。
3. 【发送前预览】发送弹窗里加了「预览（发送后的大致效果）」：
   - 按当前分组决策渲染成 Telegram 风格的气泡（靠右，跟自己发的消息一致）。
   - 图片（作为照片/相册时）显示缩略图（相册是缩略图宫格）；caption 用和聊天记录一样的 markdown 渲染；显示文件名 + 大小；相册气泡显示文件列表、总大小、>10 时提示自动分组。
   - 改 caption、切换相册/全部作文件、改单文件类型、删文件都会实时刷新预览。缩略图用 object URL，重渲染/关弹窗时 revoke 回收。
   - 改动：`app/web/ui/index.html`（预览容器）、`app/web/ui/app.js`（`renderSendPreview` 等）、`app/web/ui/style.css`（`.send-preview` / `.pv-thumb`，弹窗加 max-height 可滚）。
4. 【移除手填目标】去掉了搜索框「手填列表外 id/@用户名 直接发」的功能：现在搜索框只过滤已缓存的会话，没匹配到就显示「没有匹配的会话」，不再冒出「发送到〈输入〉」入口。
   - 改动：`app/web/ui/app.js` 删掉 `renderChatList` 里生成 manual 会话那段，并清掉所有 `kind==="manual"` 死分支（`chatKey`/`chatColorKey`/`taskMatchesChat`/`typeLabel`/`chatRow` 子行/两处 to 解析）；`index.html` 搜索框 placeholder 改成「搜索会话」。
5. 【切账号时发送中消息左右错位】群 / 频道 id 对所有账号相同，历史渲染缓存如果只按会话 key 和任务列表判断，会在切账号时复用旧 DOM，导致发送中消息仍按上一个账号靠右显示；把当前 namespace 纳入历史缓存签名后，切账号会强制重渲染，左右位置恢复正确。
6. 【发送前补充文件】发送弹窗左下角增加「＋ 添加文件」，用于已经打开弹窗后继续追加文件；追加时按 `name+size+mtime` 去重，不替换原列表，避免手滑重选导致前面配置丢失。

#### 跨账号会话记录一致 + 无权限发送提示（2026-06-16）
1.【私聊记录双向同步】以前 A 发文件给 B（私聊），记录只挂在 A 名下；切到 B 打开和 A 的对话是空的。
   - 根因：上传任务用【目标】的 id 当 chat_id（A→B 时 chat_id=B），而 B 视角里这条会话是用【对方=A】的 id 标识的，两边永远对不上。
   - 修复（纯前端 app.js）：`taskMatchesChat` 增加「收到」匹配——私聊里 chat_id==我自己的 user id 且发送方正是会话对端时，也算这条会话的记录。于是 A→B 在 B 与 A 的会话里靠左显示，B→A 在 A 那边靠左显示，双向一致。
   - 若 B 还没缓存过 A（没刷新过会话列表），`receivedChats()` 会按发送方【合成一条会话行】，让"和 A 的对话"直接冒出来；之后刷新出真实会话（同 id）自然合并、头像不变色。
   - 依赖每个账号的 self id（来自 `/accounts` 的 `web:self` 缓存，登录或点 ⟳ 后才有）。纯 CLI 建、从没在网页点过 ⟳ 的账号 self 缺失则同步不生效，点一次 ⟳ 即可。
   - 频道/群组本来就一致（群/频道 id 对所有成员相同），不用改。
2.【无权限发送：主动检查 + 失败兜底】
   - 主动：刷新会话列表时就从 dialog 自带的权限位算好能不能发——广播频道要发帖权（PostMessages 或创建者）；群看自己是否被禁言（BannedRights）和群默认权限（DefaultBannedRights）的 SendMessages/SendMedia。无权限的会话：右侧用红字横幅「🚫 无权限发送：原因」替换输入框，并禁用拖拽 / 选择文件 / 路径发送。
   - 字段用 `SendForbidden`（"禁发"，默认 false=可发），所以【老缓存 / CLI 不受影响、不会误判为禁发】；但权限刚变动要先【刷新会话列表】结果才准。
   - 私聊主要靠兜底：Telegram 不暴露"对方把我拉黑了"，事前判断不出来。
   - 兜底：真实发送失败时把 `CHAT_SEND_MEDIA_FORBIDDEN` / `CHAT_WRITE_FORBIDDEN` / `USER_IS_BLOCKED` 等错误码翻译成「无权限发送（错误码）」，并在气泡里红字内联显示（不用悬停看 tooltip）。
   - 改了哪些文件：`app/chat/ls.go`（Dialog 加 SendForbidden/SendForbidReason + 权限计算）、`app/web/errs.go`（新，错误翻译 sendErrorDisplay）、`app/web/progress.go` + `upload.go`（套用翻译）、`app/web/ui/{index.html, app.js, style.css}`（横幅 + 拦截 + 内联错误）。
   - `go build ./...` 退出码 0。前端 `go:embed`，改完重新 `go run . server` 生效。

#### Telegram 自定义分组（2026-06-22）
- 会话列表刷新时会额外读取 Telegram 的 `messages.getDialogFilters`，把账号自己的自定义分组返回给 web UI；旧缓存没有分组信息，所以升级后需要点一次「刷新会话列表」才会生成 `folders` / `folder_ids`。
- 分组归属由后端按 Telegram folder 规则匹配：手动包含、分组内置顶、手动排除、联系人/非联系人、群组、频道、机器人；`exclude_archived` 会尊重，静音/已读规则当前按需求不处理。
- `InputPeerChannel` 同时代表频道和超级群，不能只靠 UI 类型判断 peer；后端保留 `peer_kind`（user/chat/channel）做匹配，前端仍显示原来的私聊/群组/频道。
- 分组内置顶来自 Telegram 的 `PinnedPeers`，只在对应 TG 分组视图里生效；旧的 web UI 本地手动置顶功能已移除，不再读写 `localStorage` 里的 `tdl_pins:*`。
- 前端分组栏隐藏横向滚动条，鼠标在分组区域上滚轮会转成横向滚动；分组较多时不用占额外高度。

### 注意
- 只监听 `127.0.0.1`、API 无鉴权，别对外开端口。
- 同一时间别再开别的 tdl 进程：bolt 数据库是独占锁，会冲突。
- 登录仍只支持验证码方式；QR / tdata 导入、下载/转发还没做。【网页内实时上传百分比已做】（进度圆环）。【文件组/相册已做】：勾“作为相册”把多文件发成一条媒体组（≤10/组，超出自动分组）。
- 任务是【内存态】：重启 server 后“聊天记录”清空。头像颜色 / 置顶存在浏览器 localStorage（换浏览器或清缓存会重置）。
- 拖拽/选择上传会先把字节存到系统临时目录（`%TEMP%\tdl-web-upload`）暂存、发完自动清理；超大文件建议走“服务器本地路径”模式省一次传输。
- `tdl server --rate 5M` 后，网页上传也自动继承全局限速（见 [08-带宽限速.md](08-带宽限速.md)）。

## 架构与实现

命令定义在 `cmd/server.go`，业务实现在 `app/web/`，已注册进 `cmd/root.go` 的工具命令组。

`server` 命令（别名 `web`）在本地启动一个轻量 HTTP 服务，把已有的上传、会话列举等业务逻辑包装成网页接口，主要用于多账号管理和网页端上传。默认监听 `127.0.0.1:8080`，可用 `--host` 与 `-P/--port` 调整。**接口无鉴权，只应监听本地，不要对外暴露。**

整体是「薄 HTTP 层 + 复用现有业务」的结构，主要模块：

- `app/web/server.go`：路由用 `gorilla/mux`，REST 接口在 `/api` 下（账号列表、self、改名/删除、登录流程、会话列举、上传、任务）；新增账号会校验 namespace 文件名合法性；前端静态页用 `go:embed` 内嵌（源文件在 `app/web/ui/`）。
- `app/web/manager.go`（`ClientManager`）：每个 namespace（账号）维持一个常驻、已授权的客户端，靠 `RunWithAuth` 在独立 goroutine 保活，HTTP 请求派发到活客户端执行；每账号用互斥锁串行化重操作避免触发 flood；删除账号时通过 `Drop` 取消 run context、释放存储占用后才删底层文件。
- `app/web/login.go`（`LoginManager`）：网页端登录只支持验证码方式，用 channel 把 gotd 阻塞式 auth flow 桥接成异步 HTTP 输入，状态为 `starting / need_code / need_password / done / error`，单次 5 分钟超时，等价于 `tdl login -T code`；成功后把账号身份缓存到 `web:self`。
- `app/web/upload.go`：把网页请求映射成 `up.Options`，复用 `app/up` 的 `up.Run`。两个入口：`POST /upload`（发送服务器本机已有路径）与 `POST /upload-files`（multipart 接收浏览器文件字节，暂存临时目录发完清理）；网页 caption 是 markdown，转成 Telegram HTML 后包成 expr 字面量塞回既有 expr 管线；支持「作为文件」强制按文档发送；另有 `POST /upload-album` 发成一条相册。每个任务在可取消 context 中运行。
- `app/web/chats.go`：调用 `app/chat` 的 `ListDialogs` 拉取会话并缓存 peer access-hash，把列表存入 `web:chats`。默认 `GET /chats` 直接返回本地缓存、不连 Telegram，`?refresh=1` 才实时拉取覆盖。会过滤已注销账号与重复的「自己」会话，并在 `Dialog` 上附带 `Bot`、`SendForbidden`/`SendForbidReason`（从账号权限与频道/群默认权限算出，零值即可发），供前端禁用无权限会话的发送。
- `app/web/settings.go`：server 级设置（上传限速、代理）持久化到保留 namespace `_web_settings`。启动时先加载设置，再创建长期客户端；已持久化的 web 设置优先于启动 flag，代理会写回 viper，限速则向上传路径注入共享 limiter。
- `app/web/progress.go` / `errs.go`：把单个上传的字节进度写进任务表（供进度圆环），`OnDone` 收到非空错误时把任务标 `error`；权限类错误码（`CHAT_SEND_MEDIA_FORBIDDEN`、`CHAT_WRITE_FORBIDDEN`、`USER_IS_BLOCKED` 等）统一翻译成「无权限发送（错误码）」回填前端。
- `app/web/tasks.go`：内存中的上传任务表，记录状态（`queued / running / done / error`）、所属会话、文件名+大小、caption、是否相册、上传百分比，并持有取消函数；`DELETE /api/tasks/{id}` 取消并移除任务。

前端在 `app/web/ui/`（`index.html` + `app.js` + `style.css`），中文界面，分为：

- **顶部账号栏**：列出所有已登录账号（各 namespace），整行点击切换当前账号，行内可查看/刷新、改名、删除；底部「添加账号 / 登录」弹出登录向导。当前账号用 `localStorage` 记忆并驱动上传。
- **会话标签**：Telegram 式控制台。左侧会话列表（搜索框只过滤已缓存会话，Saved Messages 置顶，右键可置顶/复制 ID/刷新）；右侧把发往该会话的上传任务渲染成「只发不收」的聊天气泡，当前账号发的靠右、其他账号靠左，显示文件名/caption/大小/时间/状态，上传中显示进度圆环 + ✕ 取消。发送支持拖文件或选择文件（可勾「作为相册」、可逐个选照片/视频/文件，带发送效果预览），也支持「服务器本地路径」模式。会话记录在本机多账号间通过 `chat_id` 与各账号 `web:self` id 保持一致（A 发给 B 的文件在双方会话里对称出现）。无权限发送做了事前拦截（红字横幅替换输入框）+ 事后兜底（后端翻译权限错误回填气泡）。
- **设置弹窗**：左下角 ⚙，面向 server 全局设置而不是当前账号。上传限速热更新；代理保存后重连所有账号；代理测试只验证到 Telegram DC 的 TCP 可达性。
- **任务**：全局总览，每秒轮询展示所有账号的上传任务状态。

登录向导（弹窗）：账号名称（可留空，留空用 Telegram 昵称）+ 手机号 → 验证码 →（按需）2FA；核验期间前端文案固定为「校验中…」避免回跳。改名、删除、错误提示统一用内置弹窗组件替代浏览器原生 `prompt/confirm/alert`。

账号「改名」采用显示别名方案：别名存在该 namespace 的 `web:alias`，不改动 namespace 本身，因此不影响 CLI 默认的 `-n default`。「删除账号」先经 `ClientManager.Drop` 断开常驻连接，再由 `pkg/kv` 的 `DeleteNamespace` 删除该 namespace 全部本地数据。账号列表只列出有 session 的 namespace。

当前限制：网页端登录仍只支持验证码方式，QR / tdata 导入、下载、转发还没做；上传任务表是内存态，重启 server 后清空；头像颜色与置顶状态存在浏览器 `localStorage`。
