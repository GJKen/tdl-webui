### 网页版备份 / 恢复（设置弹窗 ⚙）

CLI 之外，web UI 也能点按钮备份/恢复了：设置弹窗里「数据备份 / 恢复」区，「备份」直接下载 `<日期>.backup.tdl`（和 CLI 产物完全通用），「恢复」上传 `.tdl` 合并导入。对应接口 `GET /api/backup`、`POST /api/recover`。

要点（和 git/源码看不出来的）：
- **恢复是「合并 / upsert」不是「整盘替换」**：备份里有的 key 覆盖同名 key，备份里没有的账号/数据【不会被删】。想干净导入到空环境，得先删掉现有账号再恢复。
- 恢复成功后会 `DropAll()` 重连所有账号（在线账号下次用时按导入的新 session 重连；新导入的账号在账号栏出现后、点选时才懒加载起客户端）。个别账号没生效就点 ⟳ 刷新或重启 server。
- 备份文件同样【没加密】= 账号凭据，别明文外传。


## 架构与实现

### 存储设计
项目的存储抽象很薄，核心接口位于 `core/storage/storage.go`：

```go
type Storage interface {
    Get(ctx context.Context, key string) ([]byte, error)
    Set(ctx context.Context, key string, value []byte) error
    Delete(ctx context.Context, key string) error
}
```

上层通过 `pkg/kv` 管理不同命名空间和底层驱动，提供命名空间级操作 `Namespaces` / `Open` / `DeleteNamespace`，bolt、file、legacy 三种驱动均已实现。当前默认存储是 Bolt，旧版 legacy 存储会在需要时自动迁移。

存储主要承载：Telegram session、peer 缓存、app 配置、下载断点续传状态、命名空间隔离数据。

默认 Bolt 数据库位于用户目录下的 `~/.tdl/data`（Windows 为 `C:\Users\<用户名>\.tdl\data`）。通过 `-n/--namespace` 切换命名空间，不同 namespace 的 session 与缓存互相隔离，默认值为 `default`。

网页版还有一个保留 namespace：`_web_settings`，用来存 server 级设置（当前是上传限速 `web:rate`、代理 `web:proxy`）。它不是账号，不会出现在账号列表里，登录时也禁止使用这个名字；但它仍是 KV namespace，所以 `backup` / `recover` / `migrate` 会把它和账号数据一起带走，这是预期行为。

### 数据备份与迁移
`cmd/migrate.go` 提供三个账号相关命令，底层实现在 `app/migrate`：

- `backup`：通过 `MigrateTo()` 读取全部 namespace 数据，JSON 序列化后用 zstd 压缩写入 `<日期>.backup.tdl`（`-d` 可指定文件名）。文件包含 session、peer 缓存与配置，**未加密**，应按账号凭据级别妥善保管。
- `recover`：`-f` 指定备份文件，解压反序列化后由 `MigrateFrom()` 导入当前存储。在新设备上 recover 后即为登录态，无需重新验证。
- `migrate`：`--to` 指定目标存储驱动，把当前数据迁移到另一种存储后端（如 legacy ↔ bolt）。

迁移到新设备也可直接复制 `~/.tdl/data` 文件，但需在 tdl 未运行时操作，且绑定 bolt 格式，通用性不如 `backup`/`recover`。

CLI 和 web 共用同一套压缩/序列化核心：`app/migrate` 里 `BackupTo(kv, io.Writer)` / `RecoverFrom(kv, io.Reader)` 是与存储无关的流式函数，CLI 命令和 web 的 `app/web/backup.go` 都调它，不各写一份 zstd/json。

**坑（web 备份下载）**：一开始让 zstd 编码器直接写 `http.ResponseWriter` 流式输出，结果响应体是 **0 字节**（并发编码器写到 ResponseWriter 没刷出来）。改成先把整个 dump 缓冲进 `bytes.Buffer`、带上 `Content-Length` 再一次写出就正常了——备份体积小（KB～个位数 MB），缓冲没负担，顺带还能在失败时返回干净的 JSON 错误码（流式写一旦开头就没法再改状态码）。
